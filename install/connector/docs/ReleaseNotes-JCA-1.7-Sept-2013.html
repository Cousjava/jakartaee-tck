JavaTM EE Connector Architecture Specification (JCA) Technology Compatibility Kit, Version 1.7
Release Notes, Sept 2013

Kit Contents

The JCA, Version 1.7 Technology Compatibility Kit (TCK) includes the following items:

  • JCA TCK tests for signature, API, and End-to-End tests:

    • A signature test checks that all of the public APIs are supported in the JCA Version 1.7 implementation that is being tested

    • API tests for all of the public APIs under the javax.resource package (and its sub-packages)

    • End-To-End tests that demonstrate compliance with the JCA 1.7 Specification

Platform Notes

The JCA TCK tests have been built and tested with JavaTM Platform, Standard Edition 7 (Java SE 7).

The JCA TCK tests have been run on the following platforms:

  • Solaris 11 Express (x64 architecture)
  • MAC OS X Mountain Lion (10.8.1+)
  • Windows XP, Windows 7, Windows 8
  • Oracle Linux 6
  • RedHat Enterprise Linux 6
  • Ubuntu Linux 12.10
  • Suse Enterprise Linux 11

The JCA TCK tests have been run against the JCA 1.7 Reference Implementation.

Known Problems and Workarounds

Running the JCA TCK Against Java EE RI on Windows

When configuring and running the JCA TCK against the RI on Windows, please make sure to install all related software (J2SE, the RI, the TCK, etc.) on the same drive (for example, C:). If you want to install on a drive other than C:, please be sure to change the following properties in <TS_HOME>/bin/ts.jte:

s1as.applicationRoot=c:
